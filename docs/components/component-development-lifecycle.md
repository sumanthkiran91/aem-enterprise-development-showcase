# Component Development Lifecycle

1. Review requirement and accessibility expectations.
2. Define dialog fields and content model.
3. Create `.content.xml` component metadata.
4. Create `cq:dialog/.content.xml`.
5. Create Sling Model interface and implementation.
6. Render with HTL.
7. Add ClientLib CSS and JS.
8. Test authoring, preview, publish, accessibility, and responsiveness.
